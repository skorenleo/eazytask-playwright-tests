# Install Playwright

## Step 1. Create a project folder

```bash
mkdir eazytask-playwright-tests
cd eazytask-playwright-tests
```

**What this does**

Creates a new folder for the Playwright project and opens it in Terminal.

**Expected result**

You are inside:
/Users/your_name/Desktop/eazytask-playwright-tests

# Step 2. Install Playwright

## Command

```bash
npm init playwright@latest
```

## What this command does

Creates a new Playwright project and installs all required dependencies.

---

## Installation Questions

### ✔ Do you want to use TypeScript or JavaScript?

**Recommended option:**

```text
TypeScript
```

**Reason:**

- Most companies use TypeScript with Playwright.
- Better code completion.
- Better error checking.

---

### ✔ Where to put your end-to-end tests?

**Recommended option:**

```text
tests
```

---

### ✔ Add a GitHub Actions workflow?

**Recommended option:**

```text
Yes
```

---

### ✔ Install Playwright browsers?

**Recommended option:**

```text
Yes
```

---

## Expected Result

The project should contain files similar to:

```text
eazytask-playwright-tests/
├── tests/
├── playwright.config.ts
├── package.json
├── package-lock.json
├── node_modules/
└── .github/
```

---

## Notes

- Always run this command inside your project folder.
- Wait until the installation is fully completed before continuing.

## Can I change my installation choices later?

Yes. Most installation options can be changed later.

Examples:

- GitHub Actions → Add manually later.
- Test folder → Rename or update the configuration.
- Browsers → Install later using:

```bash
npx playwright install
```

You do not need to reinstall Playwright.

# Step 4. Configure Playwright

## File

```text
playwright.config.ts
```

## Configuration

Add to the `use` section:

```ts
use: {
  baseURL: 'https://task.ezvz.app',
  screenshot: 'only-on-failure',
  video: 'retain-on-failure',
  trace: 'on-first-retry',
}
```

## What this does

- `baseURL` — allows using relative paths like `/login`.
- `screenshot` — saves screenshots for failed tests.
- `video` — keeps videos for failed tests.
- `trace` — records detailed test data on the first retry.

## Notes

Use relative paths:

```ts
await page.goto('/login');
```

# Step 5. Run Playwright Tests

## Command

```bash
npx playwright test
```

## What this command does

Runs all tests from the `tests/` folder.

Playwright may create example tests automatically during installation.

The command will run these example tests.

## Expected Result

```text
Running 6 tests using 4 workers
6 passed
```

## Notes

The project contains 2 example tests.

By default, Playwright runs them in 3 browsers:

- Chromium
- Firefox
- WebKit

```text
2 tests × 3 browsers = 6 test runs
```

`workers` are parallel processes used to run tests faster.

# Step 6. Open the HTML Report

## Command

```bash
npx playwright show-report
```

## What this command does

Opens the HTML report from the last test run.

## Expected Result

The report opens in the browser.

You can view:

- Test status
- Browser
- Duration
- Test details

## Notes

HTML reports are stored in:

```text
playwright-report/
```

Each new test run generates the latest report.

Previous reports are not stored as test history by default.

The report is local and is not a public URL.

# Step 9. Create the First EazyTask Test File

## File

```text
tests/login.spec.ts
```

## What to do

Create a new file inside the `tests/` folder:

```text
login.spec.ts
```

## Add this code

```ts
import { test, expect } from '@playwright/test';

test('should open login page', async ({ page }) => {
  await page.goto('/login');

  await expect(page).toHaveURL(/login/);
});
```

## What this does

- Opens the EazyTask login page.
- Checks that the URL contains `/login`.

## Notes

This is a very simple first test.

The goal is only to confirm that Playwright can open EazyTask.

# Step 11. Run a Test in a Visible Browser

## Command

```bash
npx playwright test tests/login.spec.ts --headed
```

## What this command does

Runs the test with a visible browser window.

## `--headed` vs `--ui`

- `--headed` — runs the test in a visible browser. Use it when you only want to watch the test.
- `--ui` — opens Playwright UI Mode. Use it to run, inspect, and debug tests.

```bash
npx playwright test --ui
```

## Notes

Without `--headed`, tests run in headless mode without a visible browser.
